# 要件定義書

**プロジェクト名：** 個人タスク管理アプリ  
**作成日：** 2026-05-11

---

## 1. システム概要

### 1.1 システム構成

```
[ユーザー（ブラウザ）]
        ↓ HTTP (localhost:8501)
[Streamlit フロントエンド]
        ↓ HTTP REST API (localhost:8000)
[FastAPI バックエンド]
        ↓
[JSONファイル]
(ローカルPC)
```

### 1.2 技術スタック

| レイヤー | 技術 | 用途 |
|---|---|---|
| フロントエンド | Streamlit | 画面表示・ユーザー操作 |
| バックエンド | FastAPI | APIエンドポイント・業務ロジック |
| データ保存 | JSONファイル | タスクデータの永続化 |
| 仮想環境 | uv | Pythonパッケージ管理 |
| バージョン管理 | Git / Sourcetree | ソースコード管理 |

## 2. 機能要件

### 2.1 タスク登録機能

- タイトル・カテゴリ・優先度を入力してタスクを登録できる
- 送信ボタン押下でバックエンドにPOSTリクエストを送信する
- タイトルが空の場合は送信できない（バリデーション）
- 登録成功後に完了メッセージを表示する

### 2.2 一覧表示機能

- 登録済みタスクをすべて一覧表示できる
- 表示項目：登録日時 / カテゴリ / 優先度 / ステータス / タイトル
- 登録日時の降順で表示する
- 各行から詳細画面に遷移できる

### 2.3 詳細表示機能

- タスクIDを指定して1件の詳細を表示する
- 表示項目：すべてのフィールドを表示する

### 2.4 完了更新機能

- 詳細画面から「完了にする」ボタンを押してステータスを更新できる
- 更新後、ステータスが「完了」に変わること
- 既に「完了」のタスクはボタンを非表示にする

---

## 3. 非機能要件

### 3.1 性能

- JSONファイルの読み書きはタスク件数が500件以下の場合に支障なく動作すること

### 3.2 信頼性

- JSONファイルが存在しない場合は空のリストとして扱い、アプリが起動できること

### 3.3 保守性

- フロントエンドとバックエンドはファイルを分離して管理すること
- JSONファイルのパスは設定値として1箇所にまとめること

## 4. 画面一覧

| 画面名 | 説明 |
|---|---|
| タスク登録画面 | タスクを入力し登録する |
| 一覧画面 | 登録済みタスクを一覧表示する |
| 詳細画面 | 特定のタスクの詳細を表示し、完了更新を行う |

---

## 5. API一覧（FastAPI）

| メソッド | パス | 機能 |
|---|---|---|
| POST | `/tasks` | タスクを登録してJSONに保存する |
| GET | `/tasks` | タスク一覧を返す |
| GET | `/tasks/{id}` | 指定IDのタスク詳細を返す |
| PATCH | `/tasks/{id}/complete` | 指定IDのタスクを完了に更新する |

---

## 6. データファイル仕様

- ファイル名：`tasks.json`
- 保存場所：`data/tasks.json`
- フォーマット：JSON配列（リスト形式）
- 文字コード：UTF-8

```json
[
  {
    "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "created_at": "2026-05-11T10:00:00+09:00",
    "title": "議事録を作成する",
    "category": "仕事",
    "priority": "高",
    "status": "未完了"
  }
]
```

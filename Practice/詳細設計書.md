## 1. API仕様

### 1.1 POST /tasks　タスク登録

**概要：** タスク情報を受け取り、JSONファイルに保存する。

**リクエスト**

```
POST http://localhost:8000/tasks
Content-Type: application/json
```

```json
{
  "title": "議事録を作成する",
  "category": "仕事",
  "priority": "高"
}
```

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| title | 文字列 | ○ | タスクのタイトル（空文字不可） |
| category | 文字列 | ○ | 仕事 / プライベート / その他 |
| priority | 文字列 | ○ | 高 / 中 / 低 |

**レスポンス（成功 201）**

```json
{
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "created_at": "2026-05-11T10:00:00+09:00",
  "title": "議事録を作成する",
  "category": "仕事",
  "priority": "高",
  "status": "未完了"
}
```

**エラーレスポンス**

| ステータス | 発生条件 |
|---|---|
| 422 | リクエストのJSON形式が不正 |

---

### 1.2 GET /tasks　タスク一覧取得

**概要：** 保存済みのタスクを全件、登録日時の降順で返す。

**リクエスト**

```
GET http://localhost:8000/tasks
```

**レスポンス（成功 200）**

```json
[
  {
    "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "created_at": "2026-05-11T10:00:00+09:00",
    "title": "議事録を作成する",
    "category": "仕事",
    "priority": "高",
    "status": "未完了"
  },
  {
    "id": "yyyyyyyy-yyyy-yyyy-yyyy-yyyyyyyyyyyy",
    "created_at": "2026-05-10T14:30:00+09:00",
    "title": "本を返却する",
    "category": "プライベート",
    "priority": "低",
    "status": "完了"
  }
]
```

- 件数が0件のときは空のリスト `[]` を返す

### 1.3 GET /tasks/{id}　タスク詳細取得

**概要：** 指定したIDのタスクを1件返す。

**リクエスト**

```
GET http://localhost:8000/tasks/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

**レスポンス（成功 200）**

```json
{
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "created_at": "2026-05-11T10:00:00+09:00",
  "title": "議事録を作成する",
  "category": "仕事",
  "priority": "高",
  "status": "未完了"
}
```

**エラーレスポンス**

| ステータス | 発生条件 |
|---|---|
| 404 | 指定したIDが存在しない |

---

### 1.4 PATCH /tasks/{id}/complete　タスク完了更新

**概要：** 指定したIDのタスクのステータスを「完了」に変更する。

**リクエスト**

```
PATCH http://localhost:8000/tasks/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/complete
```

リクエストボディなし。

**レスポンス（成功 200）**

```json
{
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "created_at": "2026-05-11T10:00:00+09:00",
  "title": "議事録を作成する",
  "category": "仕事",
  "priority": "高",
  "status": "完了"
}
```

**エラーレスポンス**

| ステータス | 発生条件 |
|---|---|
| 404 | 指定したIDが存在しない |

## 2. データモデル（Pydantic）

ファイル：`backend/models.py`

```python
from pydantic import BaseModel

class TaskRequest(BaseModel):
    title: str          # POST リクエストボディ
    category: str
    priority: str

class TaskRecord(BaseModel):
    id: str
    created_at: str
    title: str
    category: str
    priority: str
    status: str
```

| クラス | 用途 |
|---|---|
| `TaskRequest` | フロントエンドから受け取るPOSTリクエストのボディ |
| `TaskRecord` | JSONファイルに保存・APIから返却するデータ |

## 3. JSON 読み書き設計

ファイル：`backend/services/storage_service.py`

### 3.1 全件読み込み（load_all）

```
1. DATA_FILE_PATH のファイルが存在するか確認する
2. 存在しない場合は空のリスト [] を返す
3. 存在する場合は json.load() で読み込む
4. リストを返す
```

### 3.2 1件追記保存（save）

```
1. 全件読み込みを行いリストを取得する
2. 新しいレコードをリストの末尾に append する
3. json.dump() でファイルに上書き保存する（ensure_ascii=False, indent=2）
```

### 3.3 ID検索（find_by_id）

```
1. 全件読み込みを行いリストを取得する
2. id が一致するレコードを検索する
3. 見つかれば返す。見つからなければ None を返す
```

### 3.4 ステータス更新（update_status）

```
1. 全件読み込みを行いリストを取得する
2. id が一致するレコードを検索する
3. 見つからない場合は None を返す
4. 見つかった場合は status フィールドを「完了」に変更する
5. json.dump() でファイルを上書き保存する（ensure_ascii=False, indent=2）
6. 更新後のレコードを返す
```

## 4. 画面詳細設計

ファイル：`frontend/app.py`

### 4.1 サイドバー

```python
page = st.sidebar.radio("メニュー", ["タスク登録", "タスク一覧"])
```

選択した `page` の値に応じて表示する関数を切り替える。

---

### 4.2 タスク登録画面

```
st.title("タスク登録")

title    = st.text_input("タイトル")
category = st.selectbox("カテゴリ", ["仕事", "プライベート", "その他"])
priority = st.selectbox("優先度", ["高", "中", "低"])

if st.button("登録する"):
    if title.strip() == "":
        st.error("タイトルを入力してください。")
    else:
        response = POST /tasks に title, category, priority を送信
        if response.ok:
            st.success("タスクを登録しました")
        else:
            st.error("登録に失敗しました。")
```

---

### 4.3 一覧画面

```
st.title("タスク一覧")

response = GET /tasks
if response.ok:
    records = response.json()
    if len(records) == 0:
        st.info("タスクがありません。")
    else:
        各レコードについて：
            登録日時 / カテゴリ / 優先度 / ステータス / タイトルを1行表示
            [詳細を見る] ボタンを表示
            押されたらセッションステートに id を保存し、詳細画面へ遷移
else:
    st.error("一覧の取得に失敗しました。")
```

画面遷移の実装方針：`st.session_state` に選択したIDと遷移先ページ名を保存する。

```python
st.session_state["selected_id"] = record["id"]
st.session_state["page"] = "詳細"
st.rerun()
```

---

### 4.4 詳細画面

```
st.title("タスク詳細")

id = st.session_state.get("selected_id")
response = GET /tasks/{id}
if response.ok:
    data = response.json()
    st.write("登録日時：", data["created_at"])
    st.write("カテゴリ：", data["category"])
    st.write("優先度：",   data["priority"])
    st.write("ステータス：", data["status"])
    st.subheader("タイトル")
    st.write(data["title"])

    if data["status"] == "未完了":
        if st.button("✅ 完了にする"):
            response2 = PATCH /tasks/{id}/complete
            if response2.ok:
                st.rerun()
            else:
                st.error("更新に失敗しました。")

    if st.button("← 一覧に戻る"):
        st.session_state["page"] = "タスク一覧"
        st.rerun()

elif response.status_code == 404:
    st.error("指定されたタスクが見つかりません。")
else:
    st.error("詳細の取得に失敗しました。")
```

## 5. エラー処理一覧

| No | 発生箇所 | エラー内容 | 処理内容 | ユーザー表示 |
|---:|---|---|---|---|
| 1 | Streamlit（登録画面） | タイトルが空 | 登録APIを呼ばずに停止 | 「タイトルを入力してください。」 |
| 2 | FastAPI（JSON読み込み） | tasks.jsonが存在しない | 空リストとして処理を続行 | （エラーなし） |
| 3 | FastAPI（詳細取得） | 指定IDが存在しない | 404を返す | 「指定されたタスクが見つかりません。」 |
| 4 | FastAPI（完了更新） | 指定IDが存在しない | 404を返す | 「更新に失敗しました。」 |

## 6. テスト観点

### 6.1 正常系テスト

| No | テスト内容 | 確認ポイント |
|---:|---|---|
| 1 | タスクを登録する | 登録完了メッセージが表示されること |
| 2 | 登録後に一覧を表示する | 登録したタスクが一覧の先頭に表示されること |
| 3 | 一覧から詳細を表示する | 全フィールドが正しく表示されること |
| 4 | 詳細画面で「完了にする」を押す | ステータスが「完了」に変わり、ボタンが非表示になること |
| 5 | 詳細から一覧に戻る | 一覧画面に遷移すること |
| 6 | 複数件を登録する | すべての件数が一覧に表示されること |

### 6.2 異常系テスト

| No | テスト内容 | 確認ポイント |
|---:|---|---|
| 7 | タイトルを空にして登録ボタンを押す | エラーメッセージが表示され、登録されないこと |
| 8 | tasks.jsonがない状態で一覧を表示する | エラーにならず「タスクがありません」と表示されること |

**プロジェクト名：** 個人タスク管理アプリ

> この要求仕様書を読み、要件定義書・基本設計書・詳細設計書・設計図を作成してください。  

---

## 1. 目的・背景

### 1.1 背景

日々の業務でやるべきことを付箋やメモ帳に書き留めているが、管理が煩雑になり、対応漏れが発生している。

### 1.2 目的

タスクをアプリで一元管理し、登録・一覧・詳細確認・完了管理ができるシステムを開発する。

---

## 2. 対象ユーザー

| ユーザー | 役割 |
|---|---|
| 個人ユーザー | タスクを登録し、一覧・詳細を確認する |

---

## 3. 利用シーン

1. ユーザーがタスクのタイトル・カテゴリ・優先度を入力して登録する
2. 登録したタスクを一覧画面で確認する
3. タスクの詳細を確認する
4. 完了したタスクに完了フラグを立てる

---

## 4. 主要機能要求

### 4.1 必須機能

| No | 機能名 | 要求内容 |
|---:|---|---|
| 1 | タスク登録 | タイトル・カテゴリ・優先度を入力してタスクを登録できること |
| 2 | 一覧表示 | 登録済みタスクを一覧で確認できること |
| 3 | 詳細表示 | タスクの全項目を確認できること |
| 4 | 完了更新 | タスクを「完了」に更新できること |

### 4.2 タスクカテゴリ

仕事 / プライベート / その他

### 4.3 優先度

高 / 中 / 低

### 4.4 ステータス

未完了 / 完了

---

## 5. 制約条件

| 項目 | 制約内容 |
|---|---|
| 実行環境 | ローカルPC上で動作すること |
| データベース | DBは使用しない（JSONファイルで代替） |
| 開発言語 | Python |
| 画面 | Streamlit |
| API | FastAPI |
| 仮想環境 | uv |

---

## 6. スコープ外（対象外）

- 認証・ログイン機能
- マルチユーザー対応
- タスクの削除機能
- 期限・通知機能
- クラウドデプロイ

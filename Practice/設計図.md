## 1. クラス図

バックエンドのクラス構成と依存関係を示す。

```mermaid
classDiagram
    class TaskRequest {
        +str title
        +str category
        +str priority
    }

    class TaskRecord {
        +str id
        +str created_at
        +str title
        +str category
        +str priority
        +str status
    }

    class StorageService {
        -str file_path
        +load_all() list
        +save(record: dict) None
        +find_by_id(id: str) dict or None
        +update_status(id: str, status: str) dict or None
    }

    class TasksRouter {
        +post_task(request: TaskRequest) TaskRecord
        +get_tasks() list
        +get_task(id: str) TaskRecord
        +complete_task(id: str) TaskRecord
    }

    TasksRouter --> StorageService : 使用
    TasksRouter ..> TaskRequest : 受け取る
    TasksRouter ..> TaskRecord : 返す
    StorageService ..> TaskRecord : 読み書き対象
```

**クラスの役割まとめ**

| クラス | ファイル | 役割 |
|---|---|---|
| `TaskRequest` | backend/models.py | POSTリクエストのボディを受け取るモデル |
| `TaskRecord` | backend/models.py | 保存・返却するデータのモデル |
| `StorageService` | backend/services/storage_service.py | JSONファイルの読み書きを担当する |
| `TasksRouter` | backend/routers/tasks.py | APIエンドポイントを定義し、StorageServiceを呼び出す |

## 3. シーケンス図

### 3.1 タスク登録（正常系）

```mermaid
sequenceDiagram
    actor ユーザー
    participant ST as Streamlit
    participant FA as FastAPI
    participant SS as StorageService
    participant JS as tasks.json

    ユーザー->>ST: タイトル・カテゴリ・優先度を入力して「登録する」を押す
    ST->>ST: タイトルが空でないか確認する
    ST->>FA: POST /tasks {"title":"...","category":"...","priority":"..."}
    FA->>FA: id（UUID）と created_at（現在日時）を生成する
    FA->>FA: status を「未完了」に設定する
    FA->>SS: save(record)
    SS->>JS: load_all()（全件読み込み）
    JS-->>SS: リスト
    SS->>JS: 新レコードを追加して上書き保存
    FA-->>ST: TaskRecord（201 Created）
    ST-->>ユーザー: 「タスクを登録しました」を表示
```

### 3.2 一覧表示

```mermaid
sequenceDiagram
    actor ユーザー
    participant ST as Streamlit
    participant FA as FastAPI
    participant SS as StorageService
    participant JS as tasks.json

    ユーザー->>ST: サイドバーで「タスク一覧」を選択
    ST->>FA: GET /tasks
    FA->>SS: load_all()
    SS->>JS: ファイルを読み込む
    JS-->>SS: リスト
    SS-->>FA: リスト
    FA->>FA: created_at の降順でソートする
    FA-->>ST: list[TaskRecord]（200 OK）
    ST-->>ユーザー: 一覧をテーブルで表示する
```

### 3.3 タスク完了更新

```mermaid
sequenceDiagram
    actor ユーザー
    participant ST as Streamlit
    participant FA as FastAPI
    participant SS as StorageService
    participant JS as tasks.json

    ユーザー->>ST: 詳細画面で「完了にする」を押す
    ST->>FA: PATCH /tasks/{id}/complete
    FA->>SS: update_status(id, "完了")
    SS->>JS: load_all()
    JS-->>SS: リスト
    SS->>SS: id が一致するレコードの status を「完了」に変更する
    SS->>JS: ファイルを上書き保存する
    SS-->>FA: 更新後の TaskRecord
    FA-->>ST: TaskRecord（200 OK）
    ST-->>ユーザー: 詳細画面を再表示する（ボタンが非表示になる）
```

## 4. フロー図

### 4.1 StorageService 処理フロー

```mermaid
flowchart TD
    subgraph load["load_all()　全件読み込み"]
        A1([呼び出し]) --> B1{ファイルが存在するか?}
        B1 -->|存在しない| C1([空のリストを返す])
        B1 -->|存在する| D1[json.load でリストを読み込む]
        D1 --> E1([リストを返す])
    end

    subgraph save["save()　1件追記保存"]
        A2([呼び出し]) --> B2[load_all でリストを取得する]
        B2 --> C2[新レコードをリストの末尾に追加する]
        C2 --> D2[json.dump でファイルを上書き保存する]
        D2 --> E2([完了])
    end

    subgraph update["update_status()　ステータス更新"]
        A3([呼び出し]) --> B3[load_all でリストを取得する]
        B3 --> C3{一致する id があるか?}
        C3 -->|ない| D3([None を返す])
        C3 -->|ある| E3[status フィールドを更新する]
        E3 --> F3[json.dump でファイルを上書き保存する]
        F3 --> G3([更新後のレコードを返す])
    end
```

### 4.2 画面遷移フロー

```mermaid
flowchart TD
    Start([アプリ起動]) --> Menu{サイドバーの選択}

    Menu -->|タスク登録| RegPage[タスク登録画面を表示する]
    RegPage --> TitleCheck{タイトルが空か?}
    TitleCheck -->|空| RegError[エラーメッセージを表示する]
    RegError --> RegPage
    TitleCheck -->|入力あり| PostAPI[POST /tasks を送信する]
    PostAPI --> PostResult{APIの結果}
    PostResult -->|成功| ShowRegSuccess[「タスクを登録しました」を表示する]
    PostResult -->|失敗| PostError[エラーメッセージを表示する]

    Menu -->|タスク一覧| ListPage[一覧画面を表示する]
    ListPage --> GetAPI[GET /tasks を送信する]
    GetAPI --> ListCount{件数が0件か?}
    ListCount -->|0件| EmptyMsg[「タスクがありません」を表示する]
    ListCount -->|1件以上| ShowList[一覧をテーブルで表示する]
    ShowList --> DetailBtn{「詳細を見る」を押した?}
    DetailBtn -->|はい| SaveState[session_state に selected_id を保存する]
    SaveState --> DetailPage[詳細画面を表示する]
    DetailPage --> StatusCheck{ステータスが未完了か?}
    StatusCheck -->|未完了| ShowComplete[「完了にする」ボタンを表示する]
    StatusCheck -->|完了| HideComplete[「完了にする」ボタンを非表示にする]
    ShowComplete --> CompleteBtn{「完了にする」を押した?}
    CompleteBtn -->|はい| PatchAPI[PATCH /tasks/{id}/complete を送信する]
    PatchAPI --> DetailPage
    DetailPage --> BackBtn{「一覧に戻る」を押した?}
    BackBtn -->|はい| ListPage
```

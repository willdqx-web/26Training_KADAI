# 要求仕様書

**プロジェクト名：** AI問い合わせ管理アプリ  
**作成日：** 2026-05-11  
**対象研修：** AI問い合わせ管理アプリ開発研修

---

## 1. 目的・背景

### 1.1 背景

社員から総務部門への問い合わせは、メールや口頭など形式が統一されておらず、対応状況の把握や情報の蓄積が困難な状況にある。

同じ内容の問い合わせが繰り返されても過去の回答を参照する仕組みがなく、総務担当者の対応コストが高止まりしている。

### 1.2 目的

社員が総務へ問い合わせを送ると、AIが自動でカテゴリ・緊急度・回答案を生成し、問い合わせ履歴として管理できるアプリを開発する。

これにより、問い合わせの形式を統一し、過去の対応内容を蓄積・参照できる環境を整える。

---

## 2. 対象ユーザー

| ユーザー | 役割 |
|---|---|
| 社員（一般） | 問い合わせを入力し、一覧・詳細を確認する |

> 本バージョンでは総務担当者向けの管理・対応機能はスコープ外とする。

---

## 3. 利用シーン

1. 社員が総務への問い合わせ内容をアプリに入力する
2. AIが問い合わせを自動分類し、緊急度と回答案を生成する
3. 問い合わせ内容と生成結果がシステムに保存される
4. 社員が一覧画面で過去の問い合わせを確認する
5. 社員が詳細画面で問い合わせの内容と回答案を確認する

---

## 4. 主要機能要求

### 4.1 必須機能

| No | 機能名 | 要求内容 |
|---:|---|---|
| 1 | 問い合わせ入力 | 社員が問い合わせ文をテキストで入力できること |
| 2 | AIカテゴリ分類 | 入力内容をもとにAIが問い合わせカテゴリを自動判定すること |
| 3 | AI緊急度判定 | AIが緊急度を「高・中・低」の3段階で判定すること |
| 4 | AI回答案生成 | AIが問い合わせに対する一次回答案を自動生成すること |
| 5 | 履歴保存 | 問い合わせ内容・カテゴリ・緊急度・回答案をシステムに保存すること |
| 6 | 一覧表示 | 保存済みの問い合わせを一覧で表示できること |
| 7 | 詳細表示 | 個別の問い合わせ詳細を確認できること |

### 4.2 問い合わせカテゴリ

AIが判定するカテゴリは、以下の総務業務区分から最も適切な1つを選択する。

| カテゴリ |  | カテゴリ |  | カテゴリ |
|---|---|---|---|---|
| 勤怠 | | 休暇 | | 給与 |
| 経費精算 | | 社員情報変更 | | その他  | 

### 4.3 拡張機能（余裕がある場合に対応）

| 機能名 | 要求内容 |
|---|---|
| ステータス管理 | 「未対応・対応中・完了」を管理できること |
| 検索機能 | キーワードやカテゴリで問い合わせを絞り込めること |
| CSV出力 | 問い合わせ履歴をCSVでエクスポートできること |
| 回答案再生成 | AIに回答案を再生成させられること |
| 回答品質評価 | 5段階評価を記録できること |
| LLM切替 | GeminiとローカルLLM（LM Studio）を切り替えられること |

---

## 5. 非機能要求

| 種別 | 要求内容 |
|---|---|
| 実行環境 | ローカルPC上で動作すること |
| 操作性 | ブラウザから直感的に操作できること |
| セキュリティ | APIキーをソースコードに直接記述しないこと |
| データ管理 | 問い合わせ履歴を永続的に保存できること |

---

## 6. 制約条件

| 項目 | 制約内容 |
|---|---|
| 実行環境 | ローカルPC上で動作すること |
| データベース | DBは使用しない（JSONファイルで代替） |
| 開発言語 | Python |
| LLM | Gemini API（google-generativeai） |
| APIキー管理 | `.env` ファイルで管理し、ソースコードに直接記述しないこと |
| `.env` のGit管理 | `.env` はGitにコミットしないこと（`.gitignore` で除外） |

---

## 7. 前提条件

- Gemini APIキーを取得済みであること
- インターネット接続が利用できること（Gemini API使用時）
- 各開発者のローカルPC上で動作確認を行うこと

---

## 8. スコープ外（対象外）

- 総務担当者向けの管理・対応機能
- 認証・ログイン機能
- マルチユーザー同時利用時の排他制御
- クラウドへのデプロイ
- メール通知機能
- データベース（RDB/NoSQL）の使用

## 1. テスト概要

### 1.1 テストの目的

本テスト仕様書は、DesignDocsで定義した要件・設計をすべて満足していることを確認することを目的とする。

### 1.2 テスト対象

| レイヤー | 対象 |
|---|---|
| フロントエンド | Streamlit 画面操作・画面遷移・表示内容 |
| バックエンドAPI | FastAPI 各エンドポイントの入出力・エラーハンドリング |
| データ永続化 | JSONファイルへの読み書き |
| AI連携 | Gemini APIとの連携・回答品質 |

### 1.3 テスト環境前提条件

- Gemini APIキーを取得済みであること
- `.env` ファイルに `GEMINI_API_KEY` が設定済みであること
- インターネット接続が利用可能であること
- 以下のコマンドで FastAPI と Streamlit を起動済みであること

```
# ターミナル1（バックエンド）
uv run uvicorn backend.main:app --reload

# ターミナル2（フロントエンド）
uv run streamlit run frontend/app.py
```

### 1.4 テスト種別

| 種別 | 説明 |
|---|---|
| 機能テスト | 各機能要件を満たすかの確認 |
| APIテスト | APIエンドポイントの入出力・ステータスコードの確認 |
| UIテスト | 画面表示・画面遷移・バリデーションの確認 |
| エラー系テスト | 異常入力・障害時のエラーハンドリングの確認 |
| 非機能テスト | セキュリティ・データ管理・保守性の確認 |

### 1.5 合否判定基準

| 記号 | 意味 |
|---|---|
| ○ | 期待結果と一致し合格 |
| × | 期待結果と不一致で不合格 |
| △ | 条件付きで合格（備考欄に記載） |
